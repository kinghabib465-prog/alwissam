# تسليم المشروع للعيادة — عيادة الوسام

## رابط الدخول

| الاستخدام | الرابط |
|-----------|--------|
| **الموقع الرئيسي** | `https://YOUR-DOMAIN` |
| **دخول الطاقم** (منانة، سكرتيرة، أطباء) | `https://YOUR-DOMAIN/staff/login` |
| **دخول المريض** | `https://YOUR-DOMAIN/patient/login` |
| **حجز موعد (عام)** | `https://YOUR-DOMAIN/book-appointment` |

> استبدل `YOUR-DOMAIN` بالنطاق الفعلي المضبوط في الاستضافة (مثال: `alwisam.clinic.dz`).

---

## حسابات البداية (غيّروا كلمات المرور فوراً)

| الدور | البريد | كلمة المرور الافتراضية |
|------|--------|------------------------|
| **منانة فؤاد** — صاحبة العيادة / إدارة كاملة | `manana@alwisam.dz` | من ملف `.env` → `SEED_DOCTOR_SPECIALIST_PASSWORD` |
| **سمار بدر الدين** — سكرتيرة | `samar@alwisam.dz` | `SEED_SECRETARY1_PASSWORD` |
| **قعري أسامة** — طبيب عام | `wakri@alwisam.dz` | `SEED_DOCTOR_GENERAL_PASSWORD` |

يمكن الدخول أيضاً برقم الهاتف المسجّل في نفس الملف.

---

## ماذا تفعل العيادة يوم التشغيل؟

1. افتح **دخول الطاقم** وسجّل دخول بحساب **منانة**.
2. من لوحة الإدارة غيّر كلمات مرور الجميع.
3. حدّث بيانات العيادة (هاتف، عنوان) من الإعدادات إن وُجدت.
4. جرّب **نسيان كلمة المرور** — يجب أن يصل بريد (يتطلب ضبط SMTP في `.env`).
5. اطبعوا QR للمرضى من لوحة السكرتيرة بعد التأكد أن الرابط يظهر بالنطاق الصحيح.

## صلاحيات سريعة

| الدور | ماذا يستطيع؟ |
|------|--------------|
| **منانة** | كل شيء: جدولة، مالية، موظفين، أيام عمل السكرتارية، رفض المرضى |
| **سكرتيرة** | حجوزات، استقبال، مدفوعات، ملفات المرضى |
| **طبيب عام** | معاينة ومواعيد مرضاه فقط |
| **مريض** | مواعيده، ملفه، رسائل العيادة |

## دعم تقني (للمطوّر)

```bash
cd /opt/al-wisam-dental
docker compose -f docker-compose.prod.yml logs -f app    # سجلات التطبيق
docker compose -f docker-compose.prod.yml restart app    # إعادة تشغيل
docker compose -f docker-compose.prod.yml ps             # حالة الحاويات
```

**نسخ احتياطي يومي (مهم):**

```bash
docker exec alwisam-postgres pg_dump -U alwisam alwisam_dental > backup-$(date +%F).sql
```

## مشاكل شائعة

| المشكلة | الحل |
|---------|------|
| لا يعمل الدخول بعد تغيير البريد | استخدم البريد **الجديد**؛ إن غيّرته بنفسك أكّده من رابط البريد |
| نسيان كلمة المرور لا يرسل بريد | تأكد من `SMTP_*` أو `RESEND_API_KEY` في `.env` ثم `docker compose ... up -d --build` |
| الموقع لا يفتح | تحقق أن النطاق يشير لـ IP السيرفر وأن منافذ 80 و 443 مفتوحة |
| الجلسة تنتهي فور الدخول على HTTP | اضبط `COOKIE_SECURE=false` فقط إن كنت تستخدم IP بدون HTTPS |
